# **Session 8: SOx Reports, SOC Reports & SOX Compliance and Security Controls**

---

## 1. **Concept Overview**

* **SOX (Sarbanes-Oxley Act, 2002):** U.S. federal law ensuring **financial transparency**, **fraud prevention**, and **accountability** in corporations.
* **SOC Reports (System and Organization Controls):** Independent audit reports assessing **controls over security, availability, processing integrity, confidentiality, and privacy** in an organization.
* **SOX Compliance & Security Controls:** Processes and IT controls ensuring **integrity, confidentiality, and availability** of financial and operational data.

---

## 2. **Key Definitions**

| Term        | Definition                                       | Key Points                                                      |
| ----------- | ------------------------------------------------ | --------------------------------------------------------------- |
| SOX         | Sarbanes-Oxley Act, 2002                         | Financial transparency, prevents fraud, CEO/CFO accountability  |
| SOC Reports | System and Organization Controls                 | Audited assurance for internal controls over data and processes |
| ISAE 3402   | International standard for assurance engagements | Equivalent to SOC 1 for global entities                         |
| AUP         | Agreed Upon Procedures                           | Audit procedures based on ISRS 4400                             |
| ITGC        | IT General Controls                              | Basic IT controls ensuring reliability of financial systems     |

## 3. **Main Content (Organized)**

### **3.1 SOX Overview**

* **Purpose:**

  * Protect investors
  * Improve reporting accuracy
  * Increase corporate accountability
* **Applicability:**

  * US public companies
  * Foreign-listed companies
  * Auditing & accounting firms
* **Key Stakeholders:** CEO/CFO, Internal Audit, IT & Security teams, Board & Audit Committees

### **3.2 Historical Context & Evolution**

* **Corporate scandals:** Enron, WorldCom, Tyco, Adelphia
* **SOX Act signed:** 2002, initiated by Sen. Sarbanes & Rep. Oxley
* **PCAOB formation:** Public Company Accounting Oversight Board regulates auditors

| SOX Period | Focus                                          |
| ---------- | ---------------------------------------------- |
| 2002–2005  | Initial framework, high compliance burden      |
| 2006–2010  | Automation era, ERP integration                |
| 2011–2016  | Risk-based compliance, key control focus       |
| 2017–2020  | Cybersecurity alignment, threat-based controls |
| 2021–Today | Cloud, AI, continuous monitoring               |

### **3.3 Key SOX Sections**

| Section | Requirement                        |
| ------- | ---------------------------------- |
| 302     | CEO/CFO certify financial accuracy |
| 404     | Internal controls & reporting      |
| 401     | Accurate financial statements      |
| 802     | Criminal penalties for violations  |
| 409     | Real-time disclosure               |
| 103     | Audit standards authorization      |
| 301     | Audit committee requirements       |
| 806     | Whistleblower protection           |

### **3.4 SOX Lifecycle**

1. **Scoping & Planning:** Identify systems, define materiality
2. **Risk Assessment:** Evaluate financial impact, rank risks
3. **Control Testing:** Test ITGCs & application controls
4. **Reporting:** Management sign-off, auditor opinion

---

### **3.5 SOC Reports**

* **Purpose:** Assurance for customers/clients on control environment
* **Standards:**

  * SOC 1 → Financial reporting (equivalent to ISAE 3402)
  * SOC 2 → Security, availability, processing integrity, confidentiality, privacy
  * SOC 3 → Public summary report

> [!NOTE]
> **Audit Process:** Define scope → Readiness → Remediation → Evidence collection → Testing (Type II) → Fieldwork → Final report

| Feature      | SOC 1                | SOC 2              | SOC 3             |
| ------------ | -------------------- | ------------------ | ----------------- |
| Focus        | Financial controls   | Security & privacy | Public summary    |
| Audience     | Auditors, regulators | Clients, IT teams  | Customers, public |
| Depth        | Detailed             | Very detailed      | High-level        |
| Shareability | Restricted           | Restricted         | Public            |

* **Typical SOC report includes:**

  1. Auditor opinion
  2. Management assertion
  3. System description
  4. Control objectives / Trust services criteria
  5. Controls implemented
  6. Testing procedures & results
  7. Complementary user entity controls
  8. Subservice organizations (carve-out/inclusive)

  * **Type I:** Design effectiveness only
  * **Type II:** Operating effectiveness over time

### **3.6 Security & IT Controls**

* **ITGCs:** Access management, change management, IT operations
* **Access Controls:** Provisioning, de-provisioning, segregation of duties
* **Change Management:** Approvals, testing, deployment tracking
* **Monitoring:** Audit logs, alerts, reporting
* **Backup & Recovery:** Data protection, disaster recovery readiness

## 4. **Important Facts / Points for MCQs**

* SOX signed in **2002** after **Enron & WorldCom scandals**
* **SOC 1 = ISAE 3402**, focused on **financial reporting**
* **SOC 2** = tech-heavy, focused on **security & privacy**
* **SOC 3** = high-level public report
* **SOX 404** → Internal controls over financial reporting
* **SOX 302** → CEO/CFO certification
* **SOX 806** → Whistleblower protection
* **Common audit findings:** Access issues, change control failures, lack of documentation

---

## 5. **Examples**

* **Enron:** Fraudulent accounting → triggered SOX reforms
* **WorldCom:** $11B accounting fraud → bankruptcy & market instability
* **SOC Usage:** Cloud providers, SaaS, data centers, outsourcing vendors

---

## 6. **MCQ Pointers / Exam Traps**

* **SOC 1 vs SOC 2:** Focus difference (financial vs security)
* **SOC 2 vs SOC 3:** Audience & shareability (restricted vs public)
* **SOX Sections:** Numbers vs requirements (302 = CEO/CFO, 404 = internal controls, 806 = whistleblower)
* **Common confusions:** ISAE 3402 ≈ SOC 1 (international vs US)
* **Audit findings mix-up:** Access issues = ITGC, not application controls
