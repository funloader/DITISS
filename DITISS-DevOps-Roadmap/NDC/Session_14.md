## 1. **Concept Overview**

Session 14 introduces **logs, log analysis, and SIEM (Security Information and Event Management)** with focus on:

* Collecting and analyzing logs
* **Correlation of events**
* **Triggering alerts/incidents**
* Centralized security monitoring

Core idea:

> **SIEM converts raw logs into meaningful security events and actionable alerts**

---

## 2. **Key Definitions**

| Term             | Exam-Oriented Definition                              |
| ---------------- | ----------------------------------------------------- |
| Log              | Record of events generated by systems                 |
| Log Analyzer     | Tool that collects and analyzes logs                  |
| SIEM             | Platform for centralized log management & correlation |
| Log Correlation  | Linking multiple logs to detect incidents             |
| Event Triggering | Generating alerts based on correlation rules          |

A **Log Analyzer**:

* Helps in troubleshooting & security monitoring

## **B. Log**

---

### **Log ‚Äì Definition**

A **log** is:

* A chronological record of events
* Generated by systems, applications, and network devices

---

### **Common Log Sources (MCQ Favorite)**

* Operating systems
* Firewalls
* IDS / IPS
* Routers & switches
* Servers & applications

üìå *Every security device generates logs*

### **Log Characteristics**

* Timestamp
* Source
* Event type
* Severity
* Message

## **C. Introduction of SIEM**

---

### **SIEM ‚Äì Definition**

**SIEM (Security Information and Event Management)** is:

* A centralized security platform
* Combines:

  * SIM (Security Information Management)
  * SEM (Security Event Management)

üìå *SIEM provides real-time monitoring + historical analysis*

---

### **Functions of SIEM**

* Log collection
* Normalization
* Correlation
* Alerting
* Reporting
* Compliance support

---

## **D. SIEM Log Forwarding Configuration**

---

### **Log Forwarding ‚Äì Concept**

**Log forwarding**:

* Sends logs from devices to SIEM
* Enables centralized analysis

---

### **Common Log Forwarding Methods**

| Method      | Description                    |
| ----------- | ------------------------------ |
| Syslog      | Standard log transfer protocol |
| Agent-based | Software agent sends logs      |
| API-based   | Application-level integration  |

üìå *Syslog is most commonly used*

---

### **Why Log Forwarding is Important**

* Central visibility
* Faster detection
* Correlation across sources

---

## **E. SIEM Log Correlation and Event Triggering**

---

### **Log Correlation ‚Äì Definition**

**Log correlation**:

* Combines multiple log events
* Identifies patterns indicating attacks

üìå *Single log = event; multiple correlated logs = incident*

---

### **Examples of Log Correlation**

* Multiple failed logins + successful login
* Firewall alert + IDS alert
* Login from unusual location

---

### **Event Triggering**

**Event triggering**:

* Occurs when correlation rules match
* Generates:

  * Alerts
  * Notifications
  * Incident tickets

üìå *Triggers are rule-based*

---

### **Correlation Rule Components**

* Event type
* Time window
* Threshold
* Severity
