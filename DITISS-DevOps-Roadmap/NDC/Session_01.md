## 1. **Concept Overview**

**Information Security** deals with protecting **data, systems, and services** from unauthorized access, misuse, disruption, or destruction.

* Modern cyberspace (Internet, intranet, workplace networks) is **highly vulnerable**
* Threats come from:

  * External attackers
  * **Internal threats (disgruntled employees ‚Äì major risk)**
* Security focus:

  * **Prevention**
  * **Detection**
  * **Recovery**

---

## 2. **Key Definitions**

| Term                 | Definition (Exam-Ready)                                                                        |
| -------------------- | ---------------------------------------------------------------------------------------------- |
| Information Security | Protection of information and systems against unauthorized access, modification, or disruption |
| Threat               | Any action or event that can compromise security                                               |
| Vulnerability        | Weakness in design, implementation, or configuration                                           |
| Attack               | Actual attempt to violate security                                                             |
| Risk                 | Potential loss when a threat exploits a vulnerability                                          |
| QoS                  | Network techniques to manage bandwidth, delay, jitter, and packet loss                         |

### **D. What Needs to Be Secured?**

#### **1. Physical Security**

* Servers
* Power supply

üìå *Physical access = total compromise*

### **E. Vulnerability, Threat & Risk**

#### **Vulnerability**

* Weakness in:

  * Software
  * Hardware
  * Network
  * Configuration

#### **Threat**

* Potential cause of harm
* Example:

  * Malware
  * Insider misuse
  * Network attack

#### **Risk**

> **Risk = Threat + Vulnerability**

* Represents **expected loss**
* Includes:

  * Financial loss
  * Reputation loss
  * Legal consequences

---

### **F. Quality of Service (QoS)**

**QoS** ensures **performance reliability** for critical applications.

#### **Why QoS is important?**

* Voice / Video sensitive to delay
* Network congestion affects services

---

#### **QoS Parameters (High MCQ Value)**

| Parameter   | Meaning                           |
| ----------- | --------------------------------- |
| Packet Loss | Dropped packets due to congestion |
| Latency     | Time taken for packet to travel   |
| Jitter      | Variation in packet delay         |
| Bandwidth   | Maximum data capacity             |
| MOS         | Voice quality score (1‚Äì5)         |

üìå *MOS = Mean Opinion Score*

---

#### **QoS Models**

| Model                              | Key Point                    |
| ---------------------------------- | ---------------------------- |
| Best Effort                        | No priority, no guarantee    |
| Integrated Services (IntServ)      | Bandwidth reservation        |
| Differentiated Services (DiffServ) | Traffic classes & priorities |

## 4. **Important Facts / Points for MCQs**

* Biggest security weakness ‚Üí **Humans**

* QoS used in:

* MOS scale ‚Üí **1 (worst) to 5 (best)**
