## **1. Standard Operating Procedures (SOPs)**

### **Concept Overview**

* **Standard Operating Procedures (SOPs)** are **predefined, documented instructions** that describe **how forensic activities must be performed**.
* SOPs ensure **consistency, accuracy, legal compliance, and repeatability** in cyber forensics.

---

### **Objectives of SOPs**

* Maintain **forensic soundness**
* Ensure **evidence integrity**
* Support **legal admissibility**
* Reduce investigator error
* Standardize investigation process

---

### **Key Features of SOPs**

* Clearly defined steps
* Approved tools & techniques
* Roles and responsibilities
* Documentation requirements
* Compliance with laws & standards

---

### **Why SOPs are Important in Forensics**

* Prevent evidence contamination
* Maintain chain of custody
* Ensure uniform investigations
* Enable audit & review
* Support expert testimony

* SOPs are **mandatory**, not optional

## **2. Processing Crime and Incident Scenes ‚Äì Working with Windows & DOS Systems (Current)**

### **Concept Overview**

* Processing a crime or incident scene involves **identifying, collecting, preserving, and documenting digital evidence** from systems running **Windows or DOS**.
* Must follow **forensic best practices** and SOPs.

---

## **3. Steps in Processing Crime / Incident Scenes**

---

### **1. Secure the Scene**

* Restrict access to authorized personnel
* Prevent tampering or contamination
* Record initial condition of scene

üìå **MCQ Trap**

* Securing the scene is the **first step**

---

### **2. Identify the Type of System**

* Determine:

  * Windows OS (version)
  * DOS-based system
* Helps decide:

  * Tools
  * Techniques
  * Evidence sources

üìå **MCQ Trap**

* Tool selection depends on OS type

---

### **3. Document System Information**

#### **Record**

* Make, model, serial number
* OS version
* Installed software
* Hardware configuration

üìå **MCQ Trap**

* Documentation starts **before** evidence collection

---

### **4. Photography & Documentation**

* Capture:

  * System setup
  * Peripheral devices
  * Cable connections
  * External storage devices

üìå **MCQ Trap**

* Visual documentation is part of **chain of custody**

---

### **5. Power-Off & Disconnect (With Caution)**

* If powered ON:

  * Decide **live vs dead analysis**
* Disconnect power & peripherals **only if safe**

üìå **MCQ Trap**

* Improper shutdown may destroy volatile evidence

---

### **6. Forensic Imaging**

* Create **bit-by-bit forensic image**
* Use approved forensic tools
* Original media must remain **unchanged**

üìå **MCQ Trap**

* Imaging ‚â† file copy

---

### **7. Hash Verification**

* Calculate hash values:

  * Before imaging
  * After imaging
* Hashes must match

üìå **MCQ Trap**

* Hash mismatch = evidence integrity failure

---

### **8. Live System Analysis (If Required)**

#### **Collect Volatile Data**

* Running processes
* Network connections
* Open files
* System memory (RAM)

üìå **MCQ Trap**

* Volatile data is lost after shutdown

---

### **9. Data Recovery**

* Recover:

  * Deleted files
  * Hidden data
* Use forensic recovery tools

üìå **MCQ Trap**

* Recovery is done on **forensic copy**

---

### **10. Preserve & Document Findings**

* Maintain chain of custody
* Store evidence securely
* Prepare detailed logs & notes

üìå **MCQ Trap**

* Poor documentation weakens evidence in court

---

## **Important Facts / Points for MCQs**

* SOPs ensure repeatability
* Windows/DOS systems require OS-specific handling
* Imaging must be bit-by-bit
* Volatile data is time-sensitive
* Hashing validates integrity
* Documentation is continuous

---

## **MCQ Pointers / Exam Traps**

* SOPs vs policies
* File copy vs forensic image
* Live analysis vs dead analysis
* Volatile vs non-volatile data
* Documentation before vs after acquisition
* Hashing vs encryption
